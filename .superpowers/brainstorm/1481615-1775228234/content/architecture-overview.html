Архитектура D2Hub v2

System tray + HUD overlay + Admin panel

Обзор компонентов
🔲 System Tray
• Иконка в трее (статус: цвет)
• Правый клик → меню:
  - Открыть настройки
  - Роль: [Carry ▾]
  - Показать/скрыть HUD
  - Выход
• Автодетект dota2.exe
• Валидация конфига при старте
🎮 HUD Overlay (без изменений)
• Timer / NOW / NEXT / MACRO
• Warning windows
• + Build hint (новое)
  "Next: BKB ~4200g"
  вкл/выкл через настройки
• Frameless, click-through
• Показан только при Dota
⚙️ Admin Panel (Qt)
• Вкладки:
  - Тайминги (timeline)
  - Правила (rules)
  - Предупреждения (windows)
  - Macro тайминги
  - Сборка героя
  - Общие настройки
• Hot-reload YAML
• CRUD без перезапуска
Поток данных
Dota 2 →(GSI POST)→ GSI Server :4000 →(hero, items, map, player)→ AppController
AppController →(role detection)→ Role-filtered timings → HUD Overlay
AppController →(hero name)→ d2pt API →(build)→ HUD "Next item" hint
Admin Panel →(save YAML)→ AppController.reload() → HUD обновлён
Жизненный цикл
Запуск (.exe / ярлык) → Валидация конфигов → Tray icon (серый)
Поллинг dota2.exe (5 сек) → Dota найдена → Tray icon (зелёный) + GSI сервер старт
GSI данные получены → hero определён → роль предложена → HUD показан
Dota закрыта → HUD скрыт → GSI сервер стоп → Tray (серый, ожидание)
